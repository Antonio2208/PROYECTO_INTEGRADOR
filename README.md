# Paso 1: Inicio del Desarrollo de Software / Configuración del Entorno

### 1. Configuración de GitHub  
Se creó un repositorio público donde se almacenan los diagramas de flujo y el código del programa.  
GitHub permite llevar control de versiones, documentar el avance y mantener un entorno profesional de desarrollo.

### 2. Diagramas de flujo en Raptor  
Los diagramas representan la estructura lógica del sistema antes de programarlo.  
Incluyen:

- Validación de entradas  
- Control de decisiones  
- Estructuras repetitivas  
- Modelado de modos de operación  

Los archivos están ubicados en:

/diagramas/

### 3. Preparación del ambiente de desarrollo  
El entorno configurado incluyó:

- Python 3  
- Git y GitHub Desktop  
- Editor de código  
- Raptor para creación de diagramas  

Esto permitió iniciar la etapa de codificación con una base clara y bien estructurada.

### 4. Avance del código del software  
El primer avance del programa traduce la lógica definida en los diagramas de flujo a código Python, implementando:

- Estructuras condicionales  
- Validación de entradas del usuario  
- Comparación de valores  
- Ciclos y contadores  

El archivo se encuentra en:

/codigo/

# Paso 2: Desarrollo del Programa en Python

### Estructuras lógicas  
Uso de decisiones mediante if, elif, else.

### Estructuras repetitivas  
Se implementaron ciclos como:

- while  
- while True  
- Contadores para control de rondas  

### Manejo de excepciones  
Validación robusta mediante try/except para evitar fallos ante entradas inválidas.

### Comentarios  
Las partes más complejas del código incluyen comentarios que facilitan la comprensión y mantenimiento del sistema.
